# TASK ARCHIVE: Tab Yeet v1 — Full Extension Build

## SUMMARY

Tab Yeet v1 is a **greenfield** Firefox / LibreWolf **Manifest V2** extension. It queries tabs in the **current window**, applies user-configurable **regex transform rules** in order (no short-circuit), detects **duplicate URLs after transforms** for default checkbox state, formats the selection as **plain URLs** or **Markdown list items**, and copies the result to the **clipboard** with visible success/error feedback. An **options page** provides full CRUD, reorder, enable/disable, and regex validation for rules.

Default rules on first install rewrite common social hosts to embed-friendly mirrors (Twitter/X → `fixupx.com`, Instagram → `eeinstagram.com`, Reddit → `vxreddit.com`, TikTok → `vxtiktok.com`). Storage is centralized in **`lib/storage.js`** with canonical keys and first-run seeding when the rules key is missing (persisted empty array is preserved).

**Verification:** Vitest + jsdom, **51 tests** across `lib/`, `popup/`, `options/`, and integration. **QA** semantic review PASS (2026-04-03). **Reflect** completed (2026-04-03). Follow-up compatibility work: **`color-mix()` CSS fallbacks** for Firefox releases before **113** while `strict_min_version` remains **109.0**; default Instagram host updated to **eeinstagram.com** after embed-host verification.

## REQUIREMENTS

Sourced from the **project brief** and **`memory-bank/VISION.md`** (authoritative spec).

**Deliverables:**

- **`manifest.json`** — MV2; permissions `tabs`, `clipboardWrite`, `storage`; `browser_action` popup; `options_ui` (tab); icons 48 / 96.
- **Popup (`popup/`)** — Checkbox list of tabs (current window only), all checked by default except duplicates after transform; format dropdown; copy; gear to options; clipboard feedback (no `alert`).
- **Options (`options/`)** — Rule list: add, edit, delete, reorder, toggle enabled; inline invalid-regex messaging; persistence via shared storage.
- **`lib/transforms.js`** — Sequential `String.replace`-style application per rule; disabled rules skipped; invalid regex caught per rule; batch `transformUrls`.
- **`lib/formats.js`** — Map of format key → `(title, url) → string`; v1: `plain`, `markdown`; join with newline; unknown format falls back to `plain`.
- **`lib/storage.js`** — `STORAGE_KEYS`, `DEFAULT_RULES`, `loadRules` / `saveRules`, `loadFormatPreference` / `saveFormatPreference` (`plain` default).
- **Default rules** — Four social transforms as above (hosts may be updated post-release for embed reliability).
- **Icons** — `icons/icon-48.png`, `icons/icon-96.png`.

**Invariants (from plan):**

1. Rules run in list order against every URL.
2. Duplicate detection runs **after** transforms.
3. Tab order follows browser `index` (left-to-right).
4. Clipboard: `navigator.clipboard.writeText` primary, `document.execCommand('copy')` fallback; errors shown in UI.

## IMPLEMENTATION

**Approach:** TDD: pure modules first (`transforms`, `formats`, `storage`), then popup/options with mocked `browser.*` and jsdom; **`test/integration/extension-flow.test.js`** guards shared storage contract between options and popup.

**Preflight amendment:** Introduced **`lib/storage.js`** so keys, defaults, and `browser.storage.local` I/O live in one place—avoids duplicated defaults and typo-prone key strings across popup and options.

**Key layout:**

| Area | Role |
|------|------|
| `lib/storage.js` | Keys, `DEFAULT_RULES`, first-run `loadRules` seeding, format preference |
| `lib/transforms.js` | `applyTransforms`, `transformUrls` |
| `lib/formats.js` | `formats` map, `formatTabs` |
| `popup/popup.js` | Tab query, transform, dedupe, render, copy, feedback |
| `options/options.js` | Rule CRUD UI; uses storage helpers |
| `test/setup.js` | Mocks for `browser.tabs`, `browser.storage`, clipboard |

**Testability deviations from early plan stubs:**

- **`renderTabList(list, tabs, transformed)`** takes explicit arguments so tests do not depend on module singleton state.
- **Options `renderRules`** — synchronous DOM update (no unnecessary `async`).

**Clipboard in tests:** jsdom does not implement `document.execCommand` by default; the popup fallback test assigns a stub.

**Styling / compatibility:**

- **`color-mix()`** used for subtle borders in **`popup/popup.css`** and **`options/options.css`**. Because **`strict_min_version`** is **109.0** but `color-mix()` requires **Firefox 113+**, each border is preceded by an **rgba fallback**, with **`@media (prefers-color-scheme: dark)`** variants so separators remain visible on older supported versions.

**Embed hosts:** Default Instagram replacement is **`https://eeinstagram.com/`** (community-recommended; prior `ddinstagram` / `instagramez` iterations superseded by product decision and reachability checks).

## LESSONS LEARNED

**From reflection and delivery (inlined):**

- **Centralize storage early** — One module for keys, defaults, and first-run seeding keeps popup/options aligned; integration test locks the contract.
- **jsdom vs browser** — Stub **`document.execCommand`** when testing clipboard fallback paths.
- **CSS truncation** — Ellipsis on a flex parent does not always propagate to **nested block** children; apply overflow/ellipsis on the rows that actually hold text.
- **CSS `color-mix()` vs declared min version** — If **`strict_min_version`** is below browser support for a feature, either **raise the floor** or provide **parseable fallbacks**; unsupported shorthands can drop entirely and remove borders.
- **Preflight on L3 greenfield** — Catching missing cross-cutting modules (storage) before build avoids expensive rework.
- **Creative phase** — Not run: VISION + task list were sufficient; no friction from skipping it.

**Creative phase:** No creative phase documents were produced; design was driven by VISION and the task list. No options-considered table to inline—this section reflects that explicitly.
